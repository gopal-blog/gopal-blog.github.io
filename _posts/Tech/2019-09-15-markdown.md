When I went to school, I had a short contact with Markdown, and I put it down after learning a little fur, so I didn’t pay attention to it. When I needed it one day, I found that even the most basic grammar could be wrongly written, so I decided to take some time to revisit. It really started to be used to write blogs in GitHub Pages and use it to edit the project's README file. After doing a little research, I found that writing development documents, editing WeChat Public article and email content, etc. can be perfect native support, and suddenly found that Markdown is not a chicken rib. On the contrary, it can be said that Markdown has now penetrated all aspects of technology and has become the first choice for writing text files.

So first let's take a look at its 「 definition 」：

Markdown is a lightweight mark language, founded by John Gruber. It allows people to 「 write documents in a pure text format that is easy to read and write, and then convert them into valid XHTML（ or HTML） documents 」. —— Wikipedia

background
advantage
Focus on text content；

Pure text, easy to read and write, can be easily incorporated into version control；

The grammar is simple, there is no learning cost, and it is easy to make beautiful and generous typesetting at the same time as code words.

Use scene
Various code hosting platforms

Mainstream code hosting platforms, such as GitHub, GitLab, BitBucket, Coding, Gitee, etc., all support Markdown grammar. Many open source projects README, development documents, help documents, Wiki, etc. are written in Markdown.

Technology community and writing platform

StackOverflow, Nuggets, Brochures, GitBook

It is regrettable that each platform may use different languages to realize the Markdown analysis engine, or use different versions of the same analysis engine, and may have different degrees of customization and expansion, which leads to the experience of writing on different platforms using Markdown Not exactly the same. Fortunately, for some standard grammar that everyone recognizes, everyone supports it.

Code block
Support in-line code and code blocks.

Markdown：

Given an array of integers, return indices of the two numbers such that they add up to a specific target.
You may assume that each input would have exactly one solution, and you may not use the same element twice.

``` python
def twoSum(self, nums: 'List[int]', target: 'int') -> 'List[int]':
    dict = {}
    for i, v in enumerate(nums):
        another = target - v
        if another not in dict:
            dict[v] = [another, i]
        else:
            return [dict[another][1],i]
```

Embedded HTML
The purpose of Markdown's mark language is not to replace HTML, nor to invent a more convenient way to insert HTML labels. It corresponds to only a small subset of HTML labels.

For those HTML labels that cannot be matched by Markdown grammar, just write them directly using HTML.
